使用"hexo d"上传至github利用GitHub Actions部署静态网站

前言

Hexo博客的部署方式多种多样，对于大多数个人用户而言，使用 hexo d 命令配合 hexo-deployer-git 插件是最经典、最直接的方式。这种方式将本地生成的静态文件直接推送到GitHub仓库的特定分支，再由GitHub Pages提供服务，省去了配置复杂的CI/CD流程，简单且高效。

本文将优先介绍如何在Windows环境下使用Scoop安装高性能包管理器pnpm，并详细讲解如何配置SSH密钥与 _config.yml 实现无密码一键部署。

一、 环境准备：使用Scoop安装pnpm

pnpm以其快速的安装速度和节省磁盘空间的特性，成为替代npm的理想选择。在Windows平台，推荐使用Scoop进行命令行安装。

2.2 安装部署插件

核心步骤是安装 hexo-deployer-git 插件。该插件能够将Hexo生成的静态文件自动推送到Git仓库。

三、 配置 _config.yml 实现一键部署

3.1 获取仓库地址

在GitHub上创建一个仓库（例如 username.github.io）。为了后续配置SSH连接，请准备好仓库的SSH地址，格式通常为 git@github.com:username/username.github.io.git。

3.2 配置SSH Key连接GitHub

为了避免每次执行 hexo d 时都需要输入GitHub的用户名和密码（或Token），强烈建议配置SSH Key实现免密推送。

1. 检查现有密钥

打开 Git Bash 或 PowerShell，输入以下命令查看是否已有密钥：

如果看到 id_ed25519 或 id_rsa 文件，说明已有密钥，可直接跳到第3步。如果提示目录不存在，请继续第2步。

2. 生成新的SSH密钥

  1. 复制公钥内容：

    • 登录 GitHub，点击右上角头像 -> Settings。
    • 在左侧菜单找到 SSH and GPG keys。
    • 点击 New SSH key。
    • Title 随意填写（如 “My Laptop”），Key type 选择 “Authentication Key”。
    • 将刚才复制的公钥粘贴到 “Key” 文本框中，点击 Add SSH key。

3.3 修改配置文件

打开站点根目录下的 _config.yml 文件，找到末尾的 deploy 字段。请务必使用SSH格式的仓库地址：

配置项关键点：

  • repo: 这里必须填入刚才测试通过的SSH地址（以 git@github.com 开头），而非 https:// 开头的地址。
  • branch: GitHub Pages目前默认部署源为 main 分支，请根据你的仓库Settings -> Pages中的设置保持一致。

四、 部署与验证

4.1 执行部署命令

配置保存后，在Hexo根目录下执行以下命令：

  • hexo clean: 清除缓存。
  • hexo g: 生成静态文件。
  • -d: 部署到远程仓库。

由于我们配置了SSH Key，过程中应该不会弹出输入密码的提示，直接显示推送进度，直到出现 Deploy done: git 即为成功。

4.2 GitHub Pages 设置

部署完成后，登录GitHub仓库：

  1. 进入 Settings -> Pages。
  2. 确认 Build and deployment -> Source 选择的是 Deploy from a branch。
  3. 确认分支为你刚才推送的分支（如 main），目录为 / (root)。

稍等片刻，访问 https://你的用户名.github.io 即可看到博客内容。

五、 总结

通过配置SSH Key，我们解决了HTTPS方式频繁验证身份的烦恼，配合 hexo-deployer-git 插件，实现了真正意义上的本地“一键部署”。这种方式不仅安全便捷，而且是目前Hexo用户最推荐的标准化工作流。
